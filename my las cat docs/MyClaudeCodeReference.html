TypeScript Errors

What is it?

Type checking — catches mistakes before you run code.

What they looked like

Type 'string | undefined' is not assignable to type 'string' Property 'hours' does not exist on type 'Restaurant' Expected 2 arguments, but got 1

How you fixed them

You said: "Fix this error: [paste exact error message]"

Pattern for next time

1. Run `npm run dev`
2. Copy any TypeScript error
3. Tell Claude Code: "Fix this: [error]"
4. Repeat if needed

Common errors

Error Meaning
is not assignable to type Wrong data type
Property X does not exist Missing field on object
Cannot find module Missing import or file
Expected X arguments, got Y Wrong number of parameters
Next Time
  • Enable strict mode in Step 1: "Set 'strict: true' in tsconfig.json"
  • Define types before coding: Create types.ts with all business object types upfront.
